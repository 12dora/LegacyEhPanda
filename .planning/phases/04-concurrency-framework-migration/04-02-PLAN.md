  <action>
    Create `CountingStubProtocol: URLProtocol` plus a script model and factory:

    - Routing (probe-verified this session): the factory generates a per-test UUID token, stores
      the script state in a `static let registry = Mutex<[UUID: StubState]>([:])`
      (import Synchronization — NOT NSLock, NOT @unchecked Sendable; both are banned by lint),
      builds an ephemeral `URLSessionConfiguration` with `protocolClasses = [CountingStubProtocol.self]`
      and `httpAdditionalHeaders = ["X-EhPanda-Stub-Token": token.uuidString]`. Session-config
      additional headers ARE visible on the request inside the URLProtocol (verified by local
      probe on this toolchain) — the stub reads the token, looks up its state, and strips the
      token header before recording the received request so recorded requests reflect exactly
      what the Request struct assembled.
    - Script model: steps are keyed by request URL (`absoluteString`) so multi-fetch requests
      (TagTranslatorRequest's two URLs; the image fan-out's N thumbnail URLs) get independent
      per-URL step lists and attempt counters. Each URL carries an ordered `[StubStep]`;
      `.transportFailure(URLError.Code)` fails the load via `client?.urlProtocol(self, didFailWithError:)`;
      `.http(status:data:headers:)` responds with an HTTPURLResponse + data. When attempts exceed
      the step list, repeat the last step (so a single `.transportFailure` step models a
      persistently failing endpoint across retries).
    - Counting: `startLoading` increments the per-URL attempt counter before serving.
    - Body capture: record `request.httpBody`, falling back to draining `request.httpBodyStream`
      when the body arrives as a stream (probe-verified pattern) — POST-body assertions in the
      Account/GData baselines depend on this.
    - `StubHandle` exposes `attempts(for: URL) -> Int`, `receivedRequests: [URLRequest]`
      (token-stripped), and a `tearDown()` that removes the registry entry; the factory returns
      `(URLSession, StubHandle)`.
    - No force-unwraps anywhere (lint error) — use guard-lets; a missing registry entry should
      fail the load with a clear error rather than trap.
  </action>

    `func capture<T: Sendable>(_ operation: () async throws(AppError) -> T) async -> Result<T, AppError>`
    implemented with an explicit `do throws(AppError)` body (Pitfall 1: bare do/catch in closures
    binds `any Error`). Document in a doc comment WHY this takes a closure instead of a generic
    `<R: Request>` parameter: a generic adapter calling a protocol member dispatches statically to
    the protocol-extension facade even after a struct gains a concrete typed-throws method
    (verified by compile probe) — it would silently keep exercising the Combine path and fake the
    parity proof. Tests must always form the call on the CONCRETE request type inside the closure.

    HarnessSelfTests.swift (@Suite, Swift Testing): prove the harness contract with plain
    `session.data(for:)` calls (no Request structs needed):
    1. Token isolation: two stubbed sessions with different scripts for the SAME URL, exercised
       concurrently (async let), each sees only its own script and counters.
    2. Counting: a script with one transportFailure step, called once → attempts == 1; fixture
       step served → data round-trips.
    3. POST body capture: a POST with a form body arrives in receivedRequests with the body bytes
       intact and no stub-token header present.
    4. capture: a closure that throws AppError.noUpdates yields .failure(.noUpdates); a returning
       closure yields .success.
