<objective>
Convert the reader's core paging from the SwiftUIPager `Pager` (ReadingView.swift:143-155) to a stock
horizontal paging `ScrollView` (D-04/D-05) and consolidate the shared page index onto a single app-owned
plain-index source of truth (D-07), leaving the reader compile-clean and functionally paging. This wave
handles the CORE: the shared-index model, the horizontal ScrollView, the vertical `AdvancedList` re-seam
(lowest risk — already native), resume-seeding, the reducer fan-out, and freezing paging under zoom.
Programmatic-jump HARDENING (autoplay/slider/tap feedback-guard + gesture-coexistence proof) is Plan 04.

Planner decision on D-07 (Claude's discretion — shared index shape): introduce a thin `@Observable`
`PageModel` that mirrors the exact `Page` surface the reader uses (`.index`, `.update(.next)`,
`.update(.new(index:))`, `.withIndex(_:)`), rather than a bare `Int`. Rationale: the `.update(...)`/`.index`
call sites in `AdvancedList`, `ReadingView+Gestures`, `AutoPlayHandler`, and `ReadingView`'s handler
methods stay textually identical after only a type swap — a measurably smaller diff than threading a bare
`Int?` binding through five writers/readers — and it centralizes the index clamp (Page `isInfinite=false`
clamp parity, also the bounds-safe security mitigation). This is an app-owned standard type, not a
third-party replacement (D-01 preserved). `PageModel` is a new FILE in an existing module, so the
AGENTS.md per-module `.swiftlint.yml` rule does not trigger (no new module).

Purpose: prove the reader's paging + single-source-of-truth shape natively while the SwiftUIPager
dependency is still present (trivial rollback), keeping the reducer contract and PageHandler mapping
untouched (parity). Output: a compile-clean, paging reader on `PageModel`; SwiftUIPager still imported by
`ReadingView`/`AdvancedList` is now removed at these two files; the dependency is removed later (Plan 05).

Implements D-04, D-05, D-07, D-12 (keep reading DeviceUtil.isLandscape/windowW unchanged), D-10/D-11.
</objective>
