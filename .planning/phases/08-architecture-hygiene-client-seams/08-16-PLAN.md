<objective>
Close GAP-02 (HYG-01, blocker): `fetchEhProfileIndex` snapshots `let host = state.setting.galleryHost`
(`SettingReducer+Body.swift` line 189), but `fetchEhProfileIndexDone` drops it. The completion
handler `handleFetchEhProfileIndexDone` (`SettingReducer+Helpers.swift` lines 117–148) then re-reads
`state.setting.galleryHost.url` (line 125) for the `selectedProfile` cookie write, and
`createDefaultEhProfile` (`SettingReducer+Body.swift` lines 73–86) re-reads the mutable host again. A
host switch while profile verification is in flight can write the profile cookie — or create the
default profile — against a different domain.

Fix: thread the originating `GalleryHost` through `fetchEhProfileIndexDone` and `createDefaultEhProfile`
so the completion cookie write and the default-profile creation both use the request-origin host,
preserving the caller-owned-host invariant. Add a deterministic SettingReducer regression that
switches the shared host while the completion is pending and proves both side effects stay on origin.

Purpose: restore the caller-owned-host invariant on the profile lifecycle and lock it with a
regression.
Output: `fetchEhProfileIndexDone` + `createDefaultEhProfile` carry `GalleryHost`; completion and
profile-creation use it; a focused SettingFeature suite passes; the full AppPackage suite stays green.
</objective>
