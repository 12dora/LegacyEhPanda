<objective>
Close GAP-01 (HYG-01, blocker): the normal-image refetch flow snapshots `GalleryHost` at request
construction (`ReadingReducer+ImageFetch.swift` line 231) but `refetchNormalImageURLsDone` drops it,
so the completion reducer re-reads the mutable `state.setting.galleryHost` (line 255) when it calls
`cookieClient.setSkipServer(response:host:)`. A host switch while the request is in flight writes the
skipserver cookie to the wrong domain.

Fix: thread the originating `GalleryHost` through the `refetchNormalImageURLsDone` action and use
that carried host for `setSkipServer`, so an in-flight request keeps one host across its async work
(the caller-owned-host invariant established across 08-03..08-06). Add a deterministic ReadingReducer
regression that switches the shared host while the completion is pending and proves the cookie write
targets the request-origin host.

Purpose: restore the caller-owned-host invariant on the highest-frequency reader path and lock it
with a regression so it cannot silently re-break.
Output: `refetchNormalImageURLsDone` carries `GalleryHost`; the completion uses it; a focused
ReadingFeature test suite passes; the full AppPackage suite stays green.
</objective>
