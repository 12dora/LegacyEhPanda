<objective>
Seam D (D-08): promote the `DataCache.shared` singleton to a standalone `@Dependency(\.dataCache)`.
Add a `DataCacheKey: DependencyKey` whose `liveValue` is one canonical instance; the four consumers
and the system-purge observer all resolve/reference that same actor; delete `DataCache.shared` and
`ImageClient`'s two hardcoded `.shared` literals. `DataCache` remains an `actor` (type unchanged).

Purpose: injectable cache identity for tests without a process-global, while preserving the
single-instance cache coherence the reader/library/download pipeline requires. Chosen over folding
into ImageClient (which would create client→client deps and contradict DataCache's reader-pipeline
scope).

Output: DataCache is a dependency; one shared instance everywhere; build + existing image/download
suites green.
</objective>
