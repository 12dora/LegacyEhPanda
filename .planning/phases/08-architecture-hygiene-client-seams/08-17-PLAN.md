<objective>
Close GAP-03 (HYG-01, blocker): `UserDefaultsClient` stores only `setValue` as an endpoint; the read
is a generic instance method `getValue<T>(_:)` (`UserDefaultsClient.swift` lines 16–18) that always
reads `UserDefaults.standard`. So `.noop`, `.unimplemented`, and dependency overrides control writes
but never reads — `AppRouteReducer.detectClipboardURL` (`AppRouteReducer.swift` lines 130–140) reads
the process-global store even when the client is overridden.

Fix: model the remaining read (`clipboardChangeCount`) as a `@Sendable` endpoint on
`UserDefaultsClient`, matching the existing `setValue` shape, with deterministic `noop`/`unimplemented`
values, so reads route through the injected value exactly like writes. Update `detectClipboardURL` to
call the endpoint. Add a reducer test proving a single override controls both the read and the write.

Purpose: make the UserDefaults seam fully substitutable so client-layer behavior is deterministic and
testable, per the caller-owned-injection principle.
Output: the read is an injected endpoint; `detectClipboardURL` uses it; a reducer test proves both-way
substitutability; the full AppPackage suite stays green.
</objective>
