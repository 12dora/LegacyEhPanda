<objective>
Wave-0 parity guard for the reader gesture/geometry re-plumb (D-05, the highest-risk unit). Purify
`GestureHandler` so its pan-clamp/anchor/tap-zone math is a pure function of an injected
`containerSize` + gesture-supplied `location` (removing the `DeviceUtil.absWindowW/H` and
`TouchHandler.shared` reads from inside it); wire ONE `onGeometryChange` at the reader container as
the single source that writes `containerSize`; write `GestureHandlerTests` that baseline-lock the
arithmetic; and remove `PageHandler`'s `DeviceUtil.isLandscape` default, threading the flag explicitly
and extending `PageHandlerTests`.

Purpose: This is the guard-then-swap discipline from Phases 1-4 (05-VALIDATION Wave-0). It locks the
zoom/pan/tap arithmetic BEFORE 05-09 changes the gesture SOURCES (SpatialTapGesture/MagnifyGesture),
so any coordinate drift is caught by a green→red test. Runtime behavior is preserved here: the
gesture SOURCES stay the current `TapGesture`/`MagnificationGesture` reading `TouchHandler.shared`;
only the plumbing (injected size, pure methods) changes.
Output: a pure, tested `GestureHandler`; one `onGeometryChange` source; `GestureHandlerTests` +
extended `PageHandlerTests` green.
</objective>
