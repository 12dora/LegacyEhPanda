<objective>
Remove EhPanda's custom orientation-lock machinery so the OS governs rotation universally (D-08/
D-09/D-10, UIARCH-03): delete `AppOrientationMask`, the `AppDelegate`
`supportedInterfaceOrientationsFor` override, the `AppDelegateClient` orientation API (and the whole
module — its only real consumers are the orientation flow), the reading `setOrientationPortrait`
flow, the `enablesLandscape` toggle + its localized key, and the persisted `Setting.enablesLandscape`
field.

Purpose: Retire the app-level lock so rotation follows the device on every page. `App/Info.plist`
already declares all 4 orientations, so removing the override + mask hands governance to the OS with
NO plist change. `Setting.enablesLandscape` removal is decode-safe under the v1 schema freeze
(synthesized `Codable` ignores unknown keys — confirmed in `Setting.swift`).
Output: orientation machinery gone; `AppDelegateClient` module deleted; build/lint green.
</objective>
