<background>
DEP-06 spiked whether `DeprecatedAPI` (a 7-line Swift shim around the deprecated
`CFReadStreamCreateForHTTPRequest`) could be removed. Every warning-free replacement was rejected:
URLSession cannot override the reserved `Host` header (breaks the domain-fronting S1/S2 triad), and
Network.framework means a bespoke, unverifiable HTTP/TLS stack. The package is warning-free today
only because Xcode hides deprecation warnings from *external* dependencies. The Task-2 human
checkpoint selected `document-skip` (D-12): keep it external.

The user has now chosen to **inline** it anyway (DEP-06 "Candidate C"), accepting an explicit,
scoped, documented suppression as more transparent than relying on SPM's dependency-warning hiding,
and to shed the external package. Because inlining surfaces the deprecation on our own target, the
single-purpose `LegacyCFReadStream` target is compiled with `-suppress-warnings`. No DF request
semantic (S1–S7) changes — only the module boundary and import.
</background>

<must_haves>
- DEP-06's dependency is removed by inlining, not by a behavior change; DF semantics stay intact (D-14).
- The deprecation is contained: suppression applies to only the `LegacyCFReadStream` target.
- The new module is internal (not exposed as a package product) and lint-covered per AGENTS.md.
</must_haves>
