## Trust Boundaries

| Boundary | Description |
|---|---|
| External package code -> app-owned module | Imported converter logic becomes first-party compiled code. |
| Tag translation JSON -> FileClient conversion | Downloaded or imported tag metadata is decoded and optionally converted. |
| SwiftPM manifest -> dependency resolver | Removing a Git dependency changes supply-chain exposure. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-01-02-01 | Tampering | Local SwiftyOpenCC implementation | high | mitigate | Use Wave 0 fixtures as blocking parity tests before deleting the external dependency. |
| T-01-02-02 | Denial of Service | Dictionary lookup/conversion | medium | mitigate | Keep converter failure fallback behavior and cover it where practical. |
| T-01-02-SC | Tampering | SwiftPM package resolution | high | mitigate | Remove the external package through Package.swift and Package.resolved; do not add unaudited packages. |
</threat_model>
