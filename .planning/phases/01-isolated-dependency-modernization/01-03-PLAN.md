<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Vendored OpenCC/marisa/darts code -> app binary | Imported C++ code and licenses become first-party-maintained build inputs. |
| Swift SwiftyOpenCC -> copencc C API | Opaque dictionary/converter/string handles cross a language and ownership boundary. |
| Bundled `.ocd2` resources -> copencc parser | Binary dictionaries are loaded from the module bundle and interpreted by native code. |
| Tag translation JSON -> FileClient conversion | Downloaded or imported tag metadata is decoded and optionally converted. |
| SwiftPM manifest -> dependency resolver | Removing a Git dependency changes supply-chain exposure. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-01-03-01 | Tampering | Vendored engine and dictionaries | high | mitigate | Vendor only the verified engine graph, preserve required notices, and use Wave 0 fixtures as blocking parity evidence. |
| T-01-03-02 | Denial of Service | `.ocd2` loading/conversion | medium | mitigate | Open each required default/HK/TW chain through copencc in tests and preserve observable loader errors plus FileClient fallback behavior. |
| T-01-03-03 | Spoofing/Memory Safety | Swift/copencc ownership | high | mitigate | Retain opaque dependencies for converter lifetime and destroy dictionary, converter, and converted-string handles exactly once. |
| T-01-03-04 | Tampering | Shared dictionary cache | high | mitigate | Use `Synchronization.Mutex` or compiler-checked Sendable isolation; root lint blocks `NSLock`, `@preconcurrency`, and `@unchecked Sendable`. |
| T-01-03-SC | Tampering | SwiftPM target/product graph | high | mitigate | Remove the external package through Package.swift/Package.resolved, keep copencc internal, and expose only SwiftyOpenCC. |
</threat_model>
